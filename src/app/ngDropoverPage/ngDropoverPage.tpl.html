Introduction

"Dropover" - noun 1. A hovering DOM element who's inclusion into the visible realm can be programmatically toggled.
ngDropover is a feature-rich, event-driven solution for dropdowns, popovers, tooltips, or any other time you need a trigger or triggers to hide-show elements. The only dependency is AngularJS and your imaaginaaaationnnn!
  • Open and close the dropover programmatically with events
  • Built in touch support
  • Easily create multiple triggers
  • Close dropover when clicking outside directive
    • Dropover contents can be plain HTML, an ng-include, or another directive...even another ngDropover!

    Options

    Options are supplied to the directive as an object through the ng-dropover-options attribute. The options support binding and will update on the fly. We highly suggest you make use of binding for options. To do so just add the options object to your model and wrap the attribute value in curlies: ng-dropover-options="{{myOptionsObject}}". To make things easier to read, examples on this site do not use binding for options.

    Here are all the options currently supported...

    • ngDropoverId [string] (default: $scope.$id)

      Example: <div ng-dropover="ngDropoverId"><div>

      A unique identifier for the dropover. If not defined, it will default to the unique $scope.$id for the isolated directive scope.

    • position [string] (default: 'bottom-left')

      Example: ng-dropover-options="{'position':'top-center'}"
      click to expand

    • horizontalOffset [integer] (default: 0)

      Example: ng-dropover-options="{'horizontalOffset':20}"
      Allows you to set a -/+ value in pixels to move the dropover left or right from it's set position.

    • verticalOffset [integer] (default: 0)

      Example: ng-dropover-options="{'verticalOffset':-10}"
      Allows you to set a -/+ value in pixels to move the dropover up or down from it's set position.

    • triggerEvent [string] (default: 'click')

      The type of event that triggers opening/closing the contents when interacting with the ng-dropover element. The value 'none' is useful when you just want manual events to control the opening and closing.

      Values:

      • click - results in touchstart click for both open and close
      • hover - results in mouseenter : mouseleave and will also toggle on touch
      • none - no interaction with the ng-droopover element will trigger open/close

    • closeOnClickOff [boolean] (default: true)

      If true, will close the dropover when clicking anywhere outside of it.

    • groupId [string] (default: '')

      This is a string ID so that you have the option group dropovers by some indentifier. This will be added as a class to the dropover contents and passed as the second parameter on the ngDropover.opening and ngDropover.closing events. Useful for defining CSS or hooking into a set of dropover instances.

    ngDropoverTrigger

    We have included a helper directive to trigger directives remotely. You interact with it the same as the normal ngDropover instance, and it acts on the dropover of your choice. Use it like this:
    <div ng-dropover-trigger="{targetId: 'myDropover'}" ><div>

    Options

    • targetId required [string] (default: '')

      The ngDropoverId of the dropover instance to trigger

    • triggerEvent optional [string] (default: 'click')

      The type of event that triggers the action.

      Values:

      • click - results in touchstart click for both open and close
      • hover - results in mouseenter : mouseleave and will also toggle on touch

    • action optional [string] (default: 'toggle')

      The action that the trigger performs on the specified dropover

      values:

      • toggle - opens and closes with interaction
      • open - opens with the specified action but does not trigger a close
      • close - closes with the specified action but does not trigger an open

    Classes

    • ngdo

      Added to the directive element.

    • ngdo-open

      Added to the directive element when the dropover is open.

    • ngdo-contents

      This is a class that is added on the dropover.

    • ngdo-position

      Added to the directive element based on the 'position' option value.

    • ngdo-groupId

      Added to the directive element based on the 'groupId' option value.

    • ngdo-trigger

      Added to the directive element of an ngDropoverTrigger directive.

    Events

    ngDropover is event-based, and many of those events you can send or listen for to give you greater flexibility in your application. Here are all the events and how to use them.

    *NOTE: we highly recommend using the ngDropoverTrigger directive when applicable to avoid problems with closeOnClickOff

    • ngDropover.open

      params: required [string] ngDropoverId
      example: $rootScope.$emit('ngDropover.open', 'myDropover')
      Opens the specified dropover. Emit on the rootScope for best performance.

    • ngDropover.close

      params: required [string] ngDropoverId
      example: $rootScope.$emit('ngDropover.close', 'myDropover')
      Closes the specified dropover. Emit on the rootScope for best performance.

    • ngDropover.toggle

      params: required [string] ngDropoverId
      example: $rootScope.$emit('ngDropover.toggle', 'myDropover')
      Toggles the specified dropover. Emit on the rootScope for best performance.

    • ngDropover.closeAll

      params: optional [object] ngDropoverId
      example: $rootScope.$emit('ngDropover.open', { ngDropoverId: 'myDropover' })
      Closes all but the (optionally) specified dropover. Emit on the rootScope for best performance. Takes an object instead of a string because internally this method is used to handle document clicks.

    • ngDropover.opening

      params: [function] callback( event, dropObj )
      example:
      $rootScope.$on('ngDropover.opening', function(event, dropObj) {
      // Do something
      });
      Listen for this event if you want to perform an action on the opening of a dropover, like JavaScript animation. It is dispatched as soon as the directive's open method is called. The object param contains the following info about the dropover:
      id: The ngDropoverId of the associated dropover
      element: The DOM element of the dropovers Contents
      groupId: The groupId of the dropover, if it exists

    • ngDropover.closing

      params: [function] callback( event, dropObj )
      example:
      $rootScope.$on('ngDropover.closing', function(event, dropObj) {
      // Do something
      });
      Listen for this event if you want to perform an action on the closing of a dropOver, like JavaScript animation. It is send when the internal close call is made but after the dropovers display property is set to 'none'. The dropObj contains the following info about the dropover:
      id: the ngDropoverId of the associated dropover
      element: the DOM element of the dropovers Contents
      groupId: the groupId of the dropover, if it exists

    Using Events

    It's easy to control dropovers remotely using the supplied ngDropoverTrigger directive or events. This gives you the flexibility to open and close for more advanced triggers and situations. Also, note how both dropovers are showing on the different directives. This is because they were both given the id of "myDropover" (<div ng-dropover="myDropover"...>). Be sure to give unique ids if you don't want this behavior.

    We recommend using the ngDropoverTrigger directive as it implements touch support and handles document clicks, but if you need to programatically interact with a dropover the syntax is as follows:
    $scope.$emit('ngDropover.open', 'myDropover');

    Animation Station

    With ngDropover, we leave animations and transitions up to you. We recommend one of two ways.

    1. CSS3

    All you need to do is define your css3 transitions on .ngdo-contents or [ng-dropover-contents], and then use the .ngdo-open class to define state change.

    Example:
    .ngdo-contents {
    transition: all ease-in-out 0.5s;
    opacity: 0;
    }     .ngdo-open .ngdo-contents {
    opacity: 1;
    }

    2. JavaScript

    By listening for the ngDropover.opening and ngDropover.closing events, you can define animations on the dropover the event corresponds to

    *note : you will have to handle the display property change when closing

    jQuery Example:

    $rootScope.$on('ngDropover.opening',
    function(event, dropObj) {

    $(dropObj.element).stop().slideDown();
    });     $rootScope.$on('ngDropover.closing',
    function(event, dropObj) {

    $(dropObj.element).show();
    $(dropObj.element).stop().slideUp();
    });

    The dropObj object passed to the opening and closing functions has id, groupId, and the element properties
