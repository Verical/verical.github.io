Introduction

ngDropover is a feature-rich solution for dropdowns, popovers, tooltips, toasters, or any other time you need a trigger or triggers to hide-show an element. The only dependency is AngularJS and your imagination!
  • Open and close the dropover programmatically
  • Easily create multiple triggers
  • Close dropover when clicking outside directive EVENTS! EXAMPLE
  • Close other dropovers when opening

    • ngDropover Options

    Options are supplied to the directive as an object through the ng-dropover-options attribute. The options support binding and will update on the fly. We highly suggest you make use of binding for options. To do so just add the options object to your model and wrap the attribute value in curlies: ng-dropover-options="{{myOptionsObject}}". To make things easier to read, examples on this site do not use binding for options. Here are all the options currently supported...

    position

    This option allows you to specify where, in relation to the directive's position, you would like the dropover to appear.
    Default: 'bottom-left'
    Example: ng-dropover-options="{'position':'top-center'}"

    Available Positions

    Accepted Values

    • 'bottom-left' (can also use 'bottom')
    • 'bottom-center'
    • 'bottom-right'
    • 'top-left' (can also use 'top')
    • 'top-center'
    • 'top-right'
    • 'left-bottom' (can also use 'left')
    • 'left-center'
    • 'left-top'
    • 'right-bottom' (can also use 'right')
    • 'right-center'
    • 'right-top'

    horizontalOffset

    Example: ng-dropover-options="{'horizontalOffset':20}"
    Allows you to set a -/+ value in pixels to move the dropover left or right from it's set position.

    verticalOffset

    Example: ng-dropover-options="{'verticalOffset':-10}"
    Allows you to set a -/+ value in pixels to move the dropover up or down from it's set position.

    closeOthersOnOpen

    Example: ng-dropover-options="{'closeOthersOnOpen':true}"
    Tells the directive that you want all other open dropovers to close when any dropover os opened.

    closeOnClickOff

    Example: ng-dropover-options="{'closeOnClickOff':true}"
    Tells the directive that you want the dropover to close if you click anywhere outside of it.

    triggerEvent

    Example: ng-dropover-options="{'triggerEvent':'hover'}"
    Use this to change the event that makes the dropover open. Valid values are: 'click', 'hover', 'focus', and 'none'. The value 'none' is useful when you just want manual events to control the opening and closing.

    Getting Started

    Installation is easy as UI Bootstrap has minimal dependencies - only the AngularJS and Twitter Bootstrap's CSS are required. Note: Since version 0.13.0, UI Bootstrap depends on ngAnimate for transitions and animations, such as the accordion, carousel, etc. Include ngAnimate in the module dependencies for your app in order to enable animation.

    Install with NPM

    npm install --save ngdropover

    Manual Download

    Download the lastest version here.

    Add it to your app

    angular.module('myApp', ['ngDropover'])

    Most Basic Implementation

    $modal is a service to quickly create AngularJS-powered modal windows. Creating custom modals is straightforward: create a partial view, its controller and reference them when using the service.

    The $modal service has only one method: open(options) where available options are like follows:

    • templateUrl - a path to a template representing modal's content
    • template - inline template representing the modal's content
    • scope - a scope instance to be used for the modal's content (actually the $modal service is going to create a child scope of a provided scope). Defaults to $rootScope
    • controller - a controller for a modal instance - it can initialize scope used by modal. Accepts the "controller-as" syntax in the form 'SomeCtrl as myctrl'; can be injected with $modalInstance

    Using Events

    It's easy to control dropovers remotely using events. This gives you the flexibility to open and close for more advanced triggers and situations.

    • ngDropover.open

      Opens a dropover specified by it's unique ID

      $scope.$emit('ngDropover.open', 'myDropover');
    • ngDropover.close

      Closes a dropover specified by it's unique ID

      $scope.$emit('ngDropover.close', 'myDropover');
    • ngDropover.toggle

      Toggles a dropover open/closed, specified by it's unique ID

      $scope.$emit('ngDropover.toggle', 'myDropover');
    • ngDropover.closeAll

      Closes all dropover instances, optionally leaves open one dropover specified by it's unique ID. Note, this optional dropover id must be passed as an object literal with the property name ngDropoverId instead of a string

      Close all:

      $scope.$emit('ngDropover.closeAll');

      Close all but 'myDropover'

      $scope.$emit('ngDropover.closeAll', { ngDropoverId: 'myDropover' });

    See the Pen Nqmmav by Verical (@verical) on CodePen.

    Animation Station

    With ngDropover, we leave animations and transitions up to you. We recommend one of two ways.

    1. CSS3

    All you need to do is define your css3 transitions on .ngdo-contents or [ng-dropover-contents], and then use the .ngdo-open class to define state change.

    Example:
    .ngdo-contents {
    transition: all ease-in-out 0.5s;
    opacity: 0;
    }     .ngdo-open .ngdo-contents {
    opacity: 1;
    }

    2. JavaScript

    By listening for the ngDropover.opening and ngDropover.closing events, you can define animations on the dropover the event corresponds to

    *note : you will have to handle the display property change when closing

    jQuery Example:

    $rootScope.$on('ngDropover.opening',
    function(event, dropObj) {

    $(dropObj.element).stop().slideDown();
    });     $rootScope.$on('ngDropover.closing',
    function(event, dropObj) {

    $(dropObj.element).show();
    $(dropObj.element).stop().slideUp();
    });

    The dropObj object passed to the opening and closing functions has id, group, and the element properties

    See the Pen Animation Station by Verical (@verical) on CodePen.

    Options

    *note: all options besides ngDropoverId are set via the ng-dropover-options property which takes an object literal

    • ngDropoverId [string] (default: $scope.$id)

      A unique identifier for the dropover, set via <div ng-dropover="ngDropoverId"><div>. If not defined, it will default to the unique $scope.$id for the isolated directive scope.

    • horizontalOffset, verticalOffset [integer]

      Offsets the dropover contents by the given amount in pixels

    • triggerEvent [string] (default: click)

      The type of event that triggers opening/closing the contents when interacting with the ng-dropover element

      Values:

      • mouseenter - will close on mouseleave
      • hover - results in mouseenter : mouseleave
      • click - results in touchstart click for both open and close
      • none - no interaction with the ng-droopover element will trigger open/close

    • closeOnClickOff [boolean] (default: true)

      If true, will close the dropover when clicking outside of the contents

    • position [string] (default: bottom-left)

      Positions the dropover contents relative to whatever DOM element is used as the base element for ng-dropover

        Values:
      • Offset horizontally and vertically
      • bottom-left
      • bottom-center
      • bottom-right
      • top-left
      • top-center
      • top-right
      • left-bottom
      • left-center
      • left-top
      • right-bottom
      • right-center
      • right-top

    • group [string] (default: '')

      Will be added as a class to the dropover contents, and passed as a property of dropObj on the ngDropover.opening and ngDropover.closing events. Useful for defining CSS or hooking into a set of dropover instances

    • closeOthersOnOpen [boolean] (default: true)

      When true, will close all open dropovers when the dropover is opened. This overrides closeOnClickOff: false for other dropovers.
